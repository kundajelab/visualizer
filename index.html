List up all visualizable genome data files recursivley under a specified web directory, and visualize them
(bed type files should be compressed with bgzip and indexed with tabix -p bed).
  • bam (with index .bai)
  • bigwig
  • bedGraph
  • bed (interaction, tagAlign)
  • bedpe
  • hammock (converted from narrowPeak, broadPeak and gappedPeak)
For bigwig, you don't need any conversion to visualize it, it's already compressed and indexed.
For bam, you need to have index (.bam.bai)
For bed type files (bed, bedGraph, bedpe, hammock, ...), they need to be compressed with 'bgzip' and indexed with 'tabix -p bed'.

If you don't have index for your bams and bed type files, use a converter provided in ./converter/recursive_gen_idx.sh
It will automatically and recursively bgzip/index/convert all genome data files listed above.
If you specify file extension as the second parameter, files only with that extension will be converted. 
      $ ./converter/recursive_gen_idx.sh [TARGET_DIR]
      $ ./converter/recursive_gen_idx.sh [TARGET_DIR] [FILE_EXT]

Rules-1 for track-naming/coloring/ordering (format: identifier,name,hex_color,rank).
  • identifier: Data file names with this string will have track name 'name', color 'hex_color' and rank 'rank'
  • name : Track name will be replaced. It will stack (don't leave this blank).
  • color : Track color will be overriden (leave it blank if not changing color).
  • rank (int) : Rank will stack. Tracks will be finally sorted in the ascending order of ranks (leave it blank if not sorting tracks).
Examples:
  • wgEncodeBroadHistoneGm12878H3K27me3,H3K27me3 ,#FF0000,0
  • wgEncodeHaibTfbsA549NrsfV0422111Etoh02Aln,TF-1 ,#FFFF00,1
    • wgEncodeSydhTfbsHelas3RabiggRawDataRep1,Rep1 ,,100
  • fc.signal.bw,FC ,#0000FF,10
  • IDR,IDR ,#00FFFF,20


Rules-2 for replacing track-name/color/rank generated from Rules-1 (format: org_trk_name,name,hex_color,rank).
  • org_trk_name: track name generated from rule1
  • name : Org_trk_name will be replaced. It won't stack (just replacement) (leave it blank if not changing track name)
  • color : Track color will be overriden (leave it blank if not changing color)
  • rank (int) : Rank will stack. Tracks will be finally sorted in the ascending order of ranks (leave it blank if not sorting tracks).


 

Specify track details (format: type,hex_color,name,URL)
-type: type of a track (don't leave it undefined)
  • bigwig : automatic scaling
  • bigwig1 : fixed scaling, summary method = max (min:0 ~ max:40)
  • bigwig2 : fixed scaling, summary method = max (min:0 ~ max:200)
  • bigwig3 : fixed scaling, summary method = max (min:-40 ~ max:0)
  • bam
  • interaction : arc shaped interaction between two genome coords.
      Example: 6-column bed ( chr? \t start \t end \t chr?:start-end,score \t uniq_id strand (./-/+) ) 
          chr10   81058000        81059400        chr10:81106225-81108225,0.395519        1       .
    chr10   81059400        81060000        chr10:81106225-81108225,0.097934        101     .
  • categorical15 : 15-state chromatin state map.
      Example: 4-column bed ( chr? \t start \t end \t chr_state_id (integer) ) 
          chr1    0       9800    15
    chr1    9800    10600   9
      Add the following line to the edit box below for the chr. state map of epigenome E001 and E017 (for example). 
          categorical15,E001_chr15,http://egg2.wustl.edu/roadmap/data/byFileType/chromhmmSegmentations/ChmmModels/coreMarks/jointModel/final/E001_15_coreMarks_stateno.bed.gz
    categorical15,E017_chr15,http://egg2.wustl.edu/roadmap/data/byFileType/chromhmmSegmentations/ChmmModels/coreMarks/jointModel/final/E017_15_coreMarks_stateno.bed.gz
  • categorical18 : 18-state chromatin state map.
      • Add the following line to the edit box below for the chr. state map of epigenome E004 and E050 (for example). 
          categorical18,E004_chr18,http://egg2.wustl.edu/roadmap/data/byFileType/chromhmmSegmentations/ChmmModels/core_K27ac/jointModel/final/E004_18_core_K27ac_stateno.bed.gz
    categorical18,E050_chr18,http://egg2.wustl.edu/roadmap/data/byFileType/chromhmmSegmentations/ChmmModels/core_K27ac/jointModel/final/E050_18_core_K27ac_stateno.bed.gz
  • bed : 6-column bed.
  • hammock : WashU epg. browser specific type.
-color: hex color code (look for color hex codes at here)
    bam, bed and interaction types have their own colors (according to the strand of reads or the sign of scores).

  • bigwig : blue
  • bedGraph : maroon
  • hammock : red
-name: by default track name is its filename.

IMPORTANT! : Do not leave any "undefined" types in the text area below. Specify an appropriate type for them.
track format: type,hex_color,name,URL
